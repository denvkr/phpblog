Краткая информация по логике работы проекта.
        Полностью реализовать логику работу роутинга как в описании https://symfony.com/doc/current/create_framework
        не получилось, а не получилось реализовать роутинг, возможно из-за типа настроек apache или общей конфгурации
        WAMP сервера Open Server,точно чказать пока невозможно. Пришлось делать workaround, а именно размещать разделы
        сайта в собственных директориях и настраивать в них .htaccess файлы, кроме того чтобы как то подключить роутинг 
        файл class\routingprocessor вносить изменение в корневой .htaccess чтобы все направлялось в этот файл.
        Файлы расположенные в blog(blog.php),adminblogs(adminblogs.php),templates(blogs.php) это по сути контроллеры 
        которые подготавливают и выводят необходимую информацию через одноименные шаблоны.
        Файл class\blogloader.php это по сути контроллер с всей логикой подготовки и выдачи ответов на запросы для
        вышеописанных контроллеров, под ним находятся файлы dbroutine,parseini,dbconnect.
